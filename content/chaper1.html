Các khái niệm về Spring

I. Java Spring là gì?

1. Java Spring là một "framework" sử dụng ngôn ngữ Java.

=> Để học Spring, chúng ta "bắt buộc" cần biết "cú pháp của Java".

2. Spring Boot là gì?

Spring Boot = Spring + Boot (Bootstrapping app - chạy dự án lên/start app)

  • Đơn giản hóa quá trình xây dựng Java app
  • Giảm thiểu tối đa quá trình "cấu hình" dự án
  • Đơn giản hóa quản lý thư viện và cấu hình qua Maven/Gradle
  • Cung cấp sẵn server để chạy ứng dụng
  • Không cần cài đặt server riêng lẻ

II. Cấu trúc dự án Spring

  • src: Chứa code chính của dự án
  • resources: Chứa tài nguyên như HTML, CSS, JS, file cấu hình...
  • test: Thư mục viết code test
  • target: Mã nguồn Java sau khi được dịch (JVM sẽ chạy thư mục target)
  • Gradle/Maven: Công cụ build và quản lý dependencies
  • gradlew: dành cho macOS
  • gradlew.bat: dành cho Windows
  • pom.xml (với Maven), build.gradle.kts (với Gradle): Dùng để xử lý project (compile/build & run)

2. Các khái niệm chính:

i. Spring Container

  • Thùng chứa: Thành phần cốt lõi của Spring Framework
  • Quản lý Bean: Điều phối toàn bộ vòng đời Bean
  • Dependency Injection (DI): Cung cấp phụ thuộc cho Bean
  • Inversion of Control (IoC): Container quản lý luồng ứng dụng

ii. Bean

Bean là đối tượng được định nghĩa trong Spring Container thông qua XML, Java config, hoặc annotation. (Hiểu nhanh là class trong Java ấy v:)

iv. Annotation

Annotation là cách "trang trí" class của Java để tăng sức mạnh. (decorator) Mục đích là "tăng thêm sức mạnh" cho class đấy (meta-data).
