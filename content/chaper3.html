        package com.example.buoi01;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
//@SpringBootApplication(exclude = org.springframework.boot.autoconfigure.security.servlet.SecurityAutoConfiguration.class)

	public static void main(String[] args) {
		SpringApplication.run(Buoi01Application.class, args);

Vậy chuyện gì đã xảy ra ??

Mọi người hiểu đơn giản ở chương 1 chúng ta đã cài 1 dependency tên Spring Security vào dự án (trước sau gì cũng cài thôi nên cài luôn từ đầu) và đây chính là mấu chốt.

Mặc định Spring Security sẽ bảo vệ tất cả endpoint (tức URL) ta đã config, nghĩa là phải login mới có thể truy cập và tất nhiên ta cũng có thể cấu hình được.

Việc ta thêm đoạn code dưới vào file Application.java:

@SpringBootApplication(exclude = org.springframework.boot.autoconfigure.security.servlet.SecurityAutoConfiguration.class)

là ta đã tắt hẳn Spring Security.

Spring Security khái niệm, cách hoạt động

Mọi người biết middleware không? Thì nó cũng tương tự thế.

Còn nếu không thì:

Spring Security bảo vệ hệ thống thông qua các method "đặc biệt", gọi là filter.

Thứ tự của Filter là quan trọng

Bạn hình dung nó như một chainFilter, có nghĩa là được thực hiện tuần tự, ưu tiên như sau:

  • Check bảo mật với các tấn công hay gặp: CORS, CSRF, v.v..
  • Authentication: Xác thực người dùng là ai? (ví dụ: BasicAuthenticationFilter)
  • Authorization: Người dùng có thể làm gì? (ví dụ: Authorization Filter)

Tất nhiên là nó không chỉ có thế!

Hiện tại, Spring Security đang được cấu hình mặc định (login với dữ liệu thông qua RAM).

Chúng ta cũng có thể config password và username theo ý mình bằng cách thêm đoạn code này vào file application.properties.

# config spring security 
            # spring.security.user.name=admin
            # spring.security.user.password=123456
