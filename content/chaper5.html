Thao tác xử lý dữ liệu với ORM (Object Relational Mapping)

Spring Data

Spring Data là công cụ giúp xử lý dữ liệu trong hệ sinh thái của Spring.

Mục tiêu của Spring Data là đơn giản hóa quá trình truy cập và sử dụng dữ liệu.

  • Cung cấp đối tượng Repository để truy cập database.
  • Lập trình viên không cần viết câu lệnh SQL mà vẫn có thể thao tác với database.

Cách Read/Write Data

Với database, chúng ta có 2 thao tác:

  • READ (đọc dữ liệu)
  • WRITE (update/insert/delete) – ghi/sửa đổi dữ liệu

Cơ sở dữ liệu

  • Cơ sở dữ liệu quan hệ (relational database)
  • Cơ sở dữ liệu phi quan hệ (NoSQL)

Java là một ngôn ngữ hướng đối tượng, nên mình lựa chọn cơ sở dữ liệu quan hệ (relational database).

Để thực hiện điều trên, chúng ta có 2 cách:

  • Sử dụng JDBC (Java Database Connectivity): Là cách chúng ta kiểm soát việc sử dụng database thông qua câu lệnh truy vấn SQL.
  • Sử dụng ORM (Object Relational Mapping): Là cách chúng ta truy cập/sử dụng database thông qua các object định nghĩa trong code.

Hibernate vs Spring Data JPA

Tham khảo: YouTube, Docs

Hibernate là một framework ORM trong Java. Nó giúp ánh xạ các đối tượng Java vào các bảng trong cơ sở dữ liệu quan hệ, hỗ trợ CRUD, truy vấn, và quản lý dữ liệu.

Spring Data JPA là một phần của Spring Framework, giúp trừu tượng hóa JPA để tương tác với cơ sở dữ liệu, có thể sử dụng Hibernate hoặc các JPA implementation khác.

Ở trong khoá này (và cả khoá sau nếu có) mình sẽ sử dụng cả 2 luôn nhé!
